# Project Polish

Make an existing repository look finished, credible, and attractive without changing source behavior.

## Included skill

| Skill | Purpose |
|---|---|
| `$repo-polisher` | Evidence-backed repository identity, README craft, diagrams, badges, templates, and metadata |

## Scope

`$repo-polisher` changes presentation only. It reads the project first, preserves its character, and keeps every visible claim tied to repository evidence. Application behavior, product UI auditing, release decisions, and outward-facing GitHub changes remain outside the skill or require explicit approval.

## Install this pack

Extract `dist/GiftedLoser-Project-Polish.zip` and copy the skill folder into `~/.codex/skills`. Start a fresh Codex task so automatic routing and the explicit `$repo-polisher` override use the new definition.
