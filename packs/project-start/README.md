# Project Start

Turn an idea into an approved project definition, then establish the repository safely.

## Included skills

| Skill | Purpose |
|---|---|
| `$new-idea` | Read existing material, challenge the idea honestly, resolve material gaps, and produce `docs/PROJECT.md` |
| `$project-init` | Consume the handoff when present, or initialize normally when absent |

## Workflow

```text
Idea, conversations, notes, and mockups
    → $new-idea
    → docs/PROJECT.md
    → $project-init
    → repository ready to build
```

`$new-idea` is optional. Use `$project-init` directly when the goal is already clear or the project does not need a formal handoff.

## Install this pack

Extract `dist/GiftedLoser-Project-Start.zip` and copy both skill folders into `~/.codex/skills`. Start a fresh Codex task before invoking them.
