# Project Start

Turn an idea into an approved project definition, then establish the repository safely.

## Included skills

| Skill | Purpose |
|---|---|
| `$new-idea` | Read existing material, challenge the idea honestly, resolve material gaps, and produce `docs/PROJECT.md` |
| `$project-init` | Consume the handoff when present, or initialize normally when absent |

## Workflow

```text
Idea, conversations, notes, and mockups → $new-idea → docs/PROJECT.md
Clear request to create or normalize a repo → $project-init → repository ready to build
```

`$new-idea` is optional. `READY FOR PROJECT-INIT` records a coherent handoff but does not authorize initialization or building. `$project-init` runs only when the original request includes setup work or the user asks afterward.

## Install this pack

Extract `dist/GiftedLoser-Project-Start.zip` and copy both skill folders into `~/.codex/skills`. Start a fresh Codex task so natural routing and explicit overrides use the new definitions.
